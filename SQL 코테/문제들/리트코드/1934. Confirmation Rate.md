# 개요
### [1934. Confirmation Rate](https://leetcode.com/problems/confirmation-rate/)
### 유형

### 회고

AVG를 통해 어떤 값에 대한 비율을 편리하게 구하는 방법을 알았다. 잘 써먹자
# 해결 과정

AVG 함수는 CASE WHEN 을 이용하여 어떤 값의 대한 비율을 편리하게 구할 수 있다.

`AVG(CASE WHEN ok = 'true' THEN 1 ELSE 0 END)`

예를들어 다음과 같은 표가 있고 id별로 true의 비율을 구할 경우 어떻게 될까?

| id  | ok    |
| --- | ----- |
| 1   | true  |
| 1   | false |
| 1   | true  |
| 2   | true  |
| 2   | NULL  |
| 2   | false |
| 3   | NULL  |

- id: 1의 경우 AVG 연산이 true의 개수 / 전체 개수 이렇게 계산된다. 따라서 0.66.. 이 나온다.
- id: 2의 경우 AVG 연산에서 NULL은 제외된다. 따라서 1/2 = 0.5가 나온다.
- id: 3의 경우 AVG 연산에 포함될 행이 전부 NULL이다. 따라서 계산값은 0이 아닌 NULL이 나온다.

따라서 NULL이 나오는 값을 대체하기 위해 IFNULL을 사용해 준다.

MySQL의 경우 이렇게도 사용할 수 있다.

`AVG(c.action = 'confirmed')` 

# 풀이

```sql
SELECT 
    s.user_id, 
    ROUND(IFNULL(AVG(CASE WHEN c.action = 'confirmed' THEN 1 ELSE 0 END), 0), 2) AS confirmation_rate
FROM Signups s
LEFT JOIN Confirmations c ON s.user_id = c.user_id
GROUP BY s.user_id;
```

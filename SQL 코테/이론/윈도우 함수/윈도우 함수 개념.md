윈도우함수를 써야하는 문제 프로그래머스에서 추천해줘봐.
# 윈도우 함수란?

행과 행 사이의 관계를 정의하여 계산을 수행하는 함수이다.

가장 큰 특징은 일반 집계 함수(`GROUP BY`)와 달리 행을 합치지 않고(줄이지 않고) 각 행의 상세 정보를 유지하면서 계산 결과를 추가할 수 있다.
# 기본 문법

윈도우 함수에는 `OVER` 절과 함께 사용된다.

`SELECT 함수(컬럼) OVER (PARTITION BY 컬럼 ORDER BY 컬럼) FROM 테이블명;`

- `PARTITION BY`: 데이터를 특정 그룹으로 나눈다. (생략 시 전체 데이터 대상).
- `ORDER BY`: 나누어진 그룹 내에서 계산 순서를 정한다.

# 주요 윈도우 함수 종류

| 분류             | 함수명             | 설명                    | 특징 (핵심 차이점)                  |
| -------------- | --------------- | --------------------- | ---------------------------- |
| 순위 (Ranking)   | `ROW_NUMBER()`  | 각 행에 고유한 일련번호 부여      | 중복 값이 있어도 무조건 1, 2, 3, 4     |
|                | `RANK()`        | 중복 값에 동일 순위 부여        | 중복 후 다음 순위 건너뜀 (1, 2, 2, 4)  |
|                | `DENSE_RANK()`  | 중복 값에 동일 순위 부여        | 중복 후에도 다음 순위 연속 (1, 2, 2, 3) |
|                | `NTILE(n)`      | 전체 데이터를 $n$개의 등분으로 나눔 | 4분위수 등을 구할 때 사용              |
| 집계 (Aggregate) | `SUM()`         | 윈도우 범위 내의 합계 계산       | `ORDER BY`와 쓰면 누적 합계가 됨      |
|                | `AVG()`         | 윈도우 범위 내의 평균 계산       | 부서 평균 등 그룹 통계 비교 시 유용        |
|                | `COUNT()`       | 행의 개수 계산              | 그룹 내 데이터 개수 확인               |
|                | `MAX() / MIN()` | 최대값 / 최소값 계산          | 그룹 내 최고/최저치 확인               |
| 분석 (Value)     | `LAG()`         | 현재 행 기준 이전 행의 값 가져오기  | 전날 매출, 이전 데이터 비교 시 필수        |
|                | `LEAD()`        | 현재 행 기준 다음 행의 값 가져오기  | 다음 날 매출, 미래 데이터 비교 시 필수      |
|                | `FIRST_VALUE()` | 윈도우 내 가장 첫 번째 값       | 그룹 내 최상단 데이터 추출              |
|                | `LAST_VALUE()`  | 윈도우 내 가장 마지막 값        | 그룹 내 최하단 데이터 추출              |

# 순위 관련 윈도우 함수 연습

### 예제

```sql
CREATE TABLE employees (
    id INT,
    name VARCHAR(50),
    dept VARCHAR(50),
    salary INT
);

INSERT INTO employees VALUES 
(1, 'Alice', 'Sales', 5000),
(2, 'Bob', 'Sales', 4000),
(3, 'Charlie', 'Sales', 4000),
(4, 'Charlie', 'Sales', 3000),
(5, 'David', 'Dev', 6000),
(6, 'Eve', 'Dev', 7000),
(7, 'Frank', 'Dev', 5500),
(8, 'Grace', 'HR', 4500);
```

### 각 순위 함수를 써보기

```sql
select id,name,dept,salary,
ROW_NUMBER() over (partition by dept order by salary desc) as 'ROW_NUMBER',
RANK() over (partition by dept order by salary desc) as 'RANK',
DENSE_RANK() over (partition by dept order by salary desc) as 'DENSE_RANK'
from employees
order by id;
```

![](files/Pasted%20image%2020260331133448.png)

- `RANK()` 
	- `1,2,2,4`
	- 중복이 있을 경우 같은 순위
	- 중복이 있는 만큼 다음 순위 건너뜀
- `ROW_NUMBER()` 
	- `1,2,3,4`
	- 중복이 있어도 순서대로 순위 처리
	- `order by salary, id desc` 로 처리하면 순위가 `1,3,2,4` 로 바뀜
- `DENSE_RANK()`
	- `1,2,2,3`
	- 중복이 있으면 같은 순위
	- 중복이 있어도 그냥 다음 순위로 처리

# 집계 관련 윈도우 함수 연습

### 각 집계 함수 써보기

![](files/Pasted%20image%2020260331135223.png)

```sql
SELECT 
    id, name, dept, salary,
    SUM(salary) OVER (PARTITION BY dept) as '부서_총합',
    AVG(salary) OVER (PARTITION BY dept) as '부서_평균',
    COUNT(id) OVER (PARTITION BY dept) as '부서_인원',
    MAX(salary) OVER (PARTITION BY dept) as '부서_최대'
FROM employees
ORDER BY id;
```

- `SUM(salary)` 
	- 부서별 전체 급여 합계
- `AVG(salary)`
	- 부서별 평균 급여
- `COUNT(id)`
	- 부서별 인원 수
- `MAX(salary)`
	- 부서 내에서 가장 높은 급여

### order by 를 쓰면 누적합이 된다.

![](files/Pasted%20image%2020260331135411.png)

```sql
SELECT 
    id, name, dept, salary,
    SUM(salary) OVER (PARTITION BY dept order by id) as '부서_총합',
    AVG(salary) OVER (PARTITION BY dept order by id) as '부서_평균',
    COUNT(id) OVER (PARTITION BY dept order by id) as '부서_인원',
    MAX(salary) OVER (PARTITION BY dept order by id) as '부서_최대'
FROM employees
ORDER BY id;
```

`order by` 를 붙이게 되면 각 행부터 순차적으로 처리해나간다.

- `SUM(salary) OVER (PARTITION BY dept order by id) as '부서_총합'`
	- 1행 Alice 부터 총합을 계속해서 더해나가는 방식이다.
	- `5000, 5000+4000, 5000+4000+4000, 5000+4000+4000+3000` 의 결과가 된다.

# 분석 관련 윈도우 함수 연습
### LAG, LEAD 써보기

![](files/Pasted%20image%2020260331140156.png)

```sql
SELECT 
    id, name, dept, salary,
    LAG(salary, 1) OVER (ORDER BY id) as '이전_급여',
    LEAD(salary, 1) OVER (ORDER BY id) as '다음_급여',
    LAG(name, 1) OVER (ORDER BY id) as '이전_직원명'
FROM employees
ORDER BY id;
```

- `LAG(salary,1)`
	- 이전 데이터 비교
	- 첫번째 행의 경우 비교할게 없으므로 `NULL`
- `LEAD(salary,1)`
	- 다음 데이터 비교
	- 마지막 행의 경우 비교할게 없으므로 `NULL`
- 두번째 인자
	- 두번째 인자는 몇번째 전(후) 의 행을 비교할 건지 정하는 인자
	- `LAG(salary,3)` 일 경우 3번째 전의 행과 비교

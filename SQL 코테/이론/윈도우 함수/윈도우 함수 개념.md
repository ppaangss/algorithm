윈도우함수를 써야하는 문제 프로그래머스에서 추천해줘봐.
# 윈도우 함수란?

행과 행 사이의 관계를 정의하여 계산을 수행하는 함수이다.

가장 큰 특징은 일반 집계 함수(`GROUP BY`)와 달리 행을 합치지 않고(줄이지 않고) 각 행의 상세 정보를 유지하면서 계산 결과를 추가할 수 있다.
# 기본 문법

윈도우 함수에는 `OVER` 절과 함께 사용된다.

`SELECT 함수(컬럼) OVER (PARTITION BY 컬럼 ORDER BY 컬럼) FROM 테이블명;`

- `PARTITION BY`: 데이터를 특정 그룹으로 나눈다. (생략 시 전체 데이터 대상).
- `ORDER BY`: 나누어진 그룹 내에서 계산 순서를 정한다.

# 주요 윈도우 함수 종류

| 분류             | 함수명             | 설명                    | 특징 (핵심 차이점)                  |
| -------------- | --------------- | --------------------- | ---------------------------- |
| 순위 (Ranking)   | `ROW_NUMBER()`  | 각 행에 고유한 일련번호 부여      | 중복 값이 있어도 무조건 1, 2, 3, 4     |
|                | `RANK()`        | 중복 값에 동일 순위 부여        | 중복 후 다음 순위 건너뜀 (1, 2, 2, 4)  |
|                | `DENSE_RANK()`  | 중복 값에 동일 순위 부여        | 중복 후에도 다음 순위 연속 (1, 2, 2, 3) |
|                | `NTILE(n)`      | 전체 데이터를 $n$개의 등분으로 나눔 | 4분위수 등을 구할 때 사용              |
| 집계 (Aggregate) | `SUM()`         | 윈도우 범위 내의 합계 계산       | `ORDER BY`와 쓰면 누적 합계가 됨      |
|                | `AVG()`         | 윈도우 범위 내의 평균 계산       | 부서 평균 등 그룹 통계 비교 시 유용        |
|                | `COUNT()`       | 행의 개수 계산              | 그룹 내 데이터 개수 확인               |
|                | `MAX() / MIN()` | 최대값 / 최소값 계산          | 그룹 내 최고/최저치 확인               |
| 분석 (Value)     | `LAG()`         | 현재 행 기준 이전 행의 값 가져오기  | 전날 매출, 이전 데이터 비교 시 필수        |
|                | `LEAD()`        | 현재 행 기준 다음 행의 값 가져오기  | 다음 날 매출, 미래 데이터 비교 시 필수      |
|                | `FIRST_VALUE()` | 윈도우 내 가장 첫 번째 값       | 그룹 내 최상단 데이터 추출              |
|                | `LAST_VALUE()`  | 윈도우 내 가장 마지막 값        | 그룹 내 최하단 데이터 추출              |

# 순위 관련 윈도우 함수 연습

### 예제

```sql
CREATE TABLE employees (
    id INT,
    name VARCHAR(50),
    dept VARCHAR(50),
    salary INT
);

INSERT INTO employees VALUES 
(1, 'Alice', 'Sales', 5000),
(2, 'Bob', 'Sales', 4000),
(3, 'Charlie', 'Sales', 4000),
(4, 'Charlie', 'Sales', 3000),
(5, 'David', 'Dev', 6000),
(6, 'Eve', 'Dev', 7000),
(7, 'Frank', 'Dev', 5500),
(8, 'Grace', 'HR', 4500);
```

### 각 순위 함수를 써보기

```sql
select id,name,dept,salary,
ROW_NUMBER() over (partition by dept order by salary desc) as 'ROW_NUMBER',
RANK() over (partition by dept order by salary desc) as 'RANK',
DENSE_RANK() over (partition by dept order by salary desc) as 'DENSE_RANK'
from employees
order by id;
```

![](files/Pasted%20image%2020260331133448.png)

- `RANK()` 
	- `1,2,2,4`
	- 중복이 있을 경우 같은 순위
	- 중복이 있는 만큼 다음 순위 건너뜀
- `ROW_NUMBER()` 
	- `1,2,3,4`
	- 중복이 있어도 순서대로 순위 처리
	- `order by salary, id desc` 로 처리하면 순위가 `1,3,2,4` 로 바뀜
- `DENSE_RANK()`
	- `1,2,2,3`
	- 중복이 있으면 같은 순위
	- 중복이 있어도 그냥 다음 순위로 처리

# 집계 관련 윈도우 함수 연습

### 각 집계 함수 써보기

```sql
SELECT 
    id, name, dept, salary,
    SUM(salary) OVER (PARTITION BY dept) as '부서_총합',
    AVG(salary) OVER (PARTITION BY dept) as '부서_평균',
    COUNT(id) OVER (PARTITION BY dept) as '부서_인원',
    MAX(salary) OVER (PARTITION BY dept) as '부서_최대'
FROM employees
ORDER BY id;
```

![](files/Pasted%20image%2020260331135223.png)

- `SUM(salary)` 
	- 부서별 전체 급여 합계
- `AVG(salary)`
	- 부서별 평균 급여
- `COUNT(id)`
	- 부서별 인원 수
- `MAX(salary)`
	- 부서 내에서 가장 높은 급여

### order by 를 쓰면 누적합이 된다.

```sql
SELECT 
    id, name, dept, salary,
    SUM(salary) OVER (PARTITION BY dept order by id) as '부서_총합',
    AVG(salary) OVER (PARTITION BY dept order by id) as '부서_평균',
    COUNT(id) OVER (PARTITION BY dept order by id) as '부서_인원',
    MAX(salary) OVER (PARTITION BY dept order by id) as '부서_최대'
FROM employees
ORDER BY id;
```

![](files/Pasted%20image%2020260331135411.png)

`order by` 를 붙이게 되면 각 행부터 순차적으로 처리해나간다.

- `SUM(salary) OVER (PARTITION BY dept order by id) as '부서_총합'`
	- 1행 Alice 부터 총합을 계속해서 더해나가는 방식이다.
	- `5000, 5000+4000, 5000+4000+4000, 5000+4000+4000+3000` 의 결과가 된다.

# 분석 관련 윈도우 함수 연습
### LAG, LEAD 써보기

```sql
SELECT 
    id, name, dept, salary,
    LAG(salary, 1) OVER (ORDER BY id) as '이전_급여',
    LEAD(salary, 1) OVER (ORDER BY id) as '다음_급여',
    LAG(name, 1) OVER (ORDER BY id) as '이전_직원명'
FROM employees
ORDER BY id;
```

![](files/Pasted%20image%2020260331140156.png)

- `LAG(salary,1)`
	- 이전 데이터 비교
	- 첫번째 행의 경우 비교할게 없으므로 `NULL`
- `LEAD(salary,1)`
	- 다음 데이터 비교
	- 마지막 행의 경우 비교할게 없으므로 `NULL`
- 두번째 인자
	- 두번째 인자는 몇번째 전(후) 의 행을 비교할 건지 정하는 인자
	- `LAG(salary,3)` 일 경우 3번째 전의 행과 비교

### FIRST_VALUE, LAST_VALUE

```sql
SELECT 
    id, name, dept, salary,
    
    FIRST_VALUE(name) OVER (
        PARTITION BY dept 
        ORDER BY salary ASC
    ) as '부서_최저연봉자',
    
    FIRST_VALUE(salary) OVER (
        PARTITION BY dept 
        ORDER BY salary ASC
    ) as '부서_최저연봉',
    
    LAST_VALUE(name) OVER (
        PARTITION BY dept 
        ORDER BY salary ASC
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) as '부서_최고연봉자',
    
    LAST_VALUE(salary) OVER (
        PARTITION BY dept 
        ORDER BY salary ASC
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) as '부서_최고연봉'
FROM employees
order by id;
```

![](files/Pasted%20image%2020260331141350.png)

- `FIRST_VALUE(name)`
	- 부서 내 급여가 가장 적은 사람 
	- 급여를 기준으로 오름차순 정렬하고 첫 번째 행을 가져오는 것이다.
	- 만약 `ORDER BY salary DESC` 로 하면 내림차순 정렬하고 첫 번째 행을 가져오므로 연봉이 가장 작은 Alice를 출력한다.
- `LAST_VALUE(name)`
	- 부서 내 급여가 가장 많은 사람
	- 급여를 기준으로 오름차순 정렬하고 마지막 행을 가져오는 것이다.
	- `ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING`
		- 윈도우 함수의 계산 범위를 지정하는 문법
		- 안 할 경우 이상하게 나올 수 있다.

# 윈도우 함수의 계산 범위를 설정하기

윈도우 함수의 계산 범위를 수동으로 지정하는 문법

## ROWS

물리적인 행(Row)을 기준으로 범위를 결정하는 방식이다.
### 문법 구조

`ROWS BETWEEN [시작 지점] AND [끝 지점]`

### 사용할 수 있는 키워드

- `UNBOUNDED PRECEDING`: 파티션의 맨 처음부터
- `UNBOUNDED FOLLOWING`: 파티션의 맨 끝까지
- `CURRENT ROW`: 현재 내가 서 있는 행
- `n PRECEDING`: 내 위로 $n$개 행까지
- `n FOLLOWING`: 내 아래로 $n$개 행까지


### LAST_VALUE를 통해 비교해보기

```sql
SELECT 
    id, name, dept, salary,
    
    LAST_VALUE(name) OVER (
        PARTITION BY dept 
        ORDER BY salary ASC
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) as '부서_최고연봉자',
    
    LAST_VALUE(salary) OVER (
        PARTITION BY dept 
        ORDER BY salary ASC
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) as '부서_최고연봉',
    
    LAST_VALUE(name) OVER (
        PARTITION BY dept 
        ORDER BY salary ASC
    ) as '부서_최고연봉자',
    
    LAST_VALUE(salary) OVER (
        PARTITION BY dept 
        ORDER BY salary ASC
    ) as '부서_최고연봉'

FROM employees
order by id;
```

![](files/Pasted%20image%2020260331143346.png)

부서가 Sales인 부분을 확인해보면 먼저 봉급을 기준으로 오름차순으로 정렬하면 이렇게 된다.

![](files/Pasted%20image%2020260331143610.png)

여기서 `LAST_VALUE(name) OVER ( PARTITION BY dept  ORDER BY salary ASC)` 를 하게되면 파티션 첫번째 행부터 마지막 행이 아닌, 자기 자신의 행까지만 수행을 하게 된다.

정확히는 `order by salary asc` 를 했으므로 자기 자신의 행 + 자신과 같은 salary의 행까지만 포함하는 (`RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW`) 를 자체적으로 수행한다.

Bob의 경우 Chao 부터 Bob과 같은 Charlie의 행까지만 수행하고, 마지막 행인 Charlie의 값을 가져오는 것이다.

그래서 `ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING` 를 추가하면 해당 파티션의 첫번째 행부터 마지막 행까지 전부를 확인하게 되서 정확한 값이 나오게 된다.


## RANGE

데이터의 '값(Value)'을 기준으로 범위를 결정하는 방식이다.

### 문법 구조

`RANGE BETWEEN [시작 지점] AND [끝 지점]`

### 예시: 나보다 급여가 1000원 이내로 차이나는 사람의 수

```sql
SELECT 
    id, name, dept, salary,
    COUNT(*) OVER (
        PARTITION BY dept 
        ORDER BY salary 
        RANGE BETWEEN 1000 PRECEDING AND 1000 FOLLOWING
    ) as '범위내_인원수'
FROM employees
ORDER BY dept, salary;
```

![](files/Pasted%20image%2020260331145224.png)

- Alice의 경우 Alice, Bob, Charlie 를 포함하므로 3명
- Bob의 경우 Alice, Bob, Charlie, Chao 를 포함하므로 3명

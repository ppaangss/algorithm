# UNION

조인은 두 테이블을 양옆으로 합치는 것이다.

유니온은 두 테이블을 위 아래로 합치는 것이다.

유니온을 쓸 때 주의점

- 두 테이블의 컬럼의 개수와 데이터 타입이 일치해야 한다.
	- 없는 컬럼은 어떻게?
	- `NULL AS '컬럼명'` 이렇게 가능
	- 강제로 칸을 맞춰야 한다.
- ORDER BY 는 6번째 순위로 제일 마지막에 된다.
	- 그래서 ORDER BY는 마지막에 딱 한 번만 하면 된다.
- UNION vs UNION ALL
	- UNION은 중복된 행을 제거한다.
	- UNION ALL은 중복 제거 없어 모든 데이터를 그대로 합친다.

# 예시코드

```sql
SELECT DATE_FORMAT(SALES_DATE, '%Y-%m-%d') AS SALES_DATE, 
       PRODUCT_ID, 
       USER_ID, 
       SALES_AMOUNT
FROM ONLINE_SALE
WHERE SALES_DATE >= '2022-03-01' AND SALES_DATE < '2022-04-01'

UNION ALL

SELECT DATE_FORMAT(SALES_DATE, '%Y-%m-%d') AS SALES_DATE, 
       PRODUCT_ID, 
       NULL AS USER_ID, -- 오프라인 데이터는 USER_ID를 NULL로 처리
       SALES_AMOUNT
FROM OFFLINE_SALE
WHERE SALES_DATE >= '2022-03-01' AND SALES_DATE < '2022-04-01'

ORDER BY SALES_DATE ASC, PRODUCT_ID ASC, USER_ID ASC;
```


# 윈도우 함수란?
원본 데이터의 행 수는 그대로 유지하면서, 각 행 옆에 계산 결과를 추가해주는 함수

비유하자면:
- 집계 함수: 전교생 몸무게를 다 더해서 '평균값' 하나만 적힌 성적표를 만드는 것.
- 윈도우 함수: 전교생 성적표를 그대로 두되, 그 옆 칸에 '반 등수'나 '전교 등수'를 한 줄씩 적어주는 것.

# OVER 절
윈도우 함수는 항상 뒤에 `OVER`가 붙는다. `OVER` 안에서 어떻게 줄을 세울지 결정한다.

```sql
SELECT 이름, 점수,
       RANK() OVER (ORDER BY 점수 DESC) AS 등수
FROM 시험결과;
```

OVER 안에 들어갈 수 있는 요소
- **`ORDER BY`**: 어떤 순서로 번호를 매길지 정한다. (성적이 높은 순서대로 등수를 매기겠다!)
- **`PARTITION BY`**: 그룹을 나눕니다. (예: `PARTITION BY 반` 이라고 쓰면 1반끼리 등수, 2반끼리 등수를 따로 매깁니다.)


# 순위 함수
`ROW_NUMBER() vs RANK() vs DENSE_RANK() `

- `ROW_NUMBER()`
	- 동점자가 있든 없든 그냥 1,2,3,4 번호를 붙인다.
	- 결과: 1등(100), 2등(100), 3등(90), 4등(80)
	- 용도: "게시판의 글 번호"처럼 중복 없이 번호를 매길 때 씁니다.
- `RANK()`
	- 동점자가 있으면 같은 등수를 주고, 그 수만큼 다음 등수를 건너뜁니다.
	- 결과: 1등(100), 1등(100), 3등(90), 4등(80)
	- 용도: 우리가 흔히 아는 "올림픽 순위" 방식입니다.
- `DENSE_RANK()`
	- 동점자가 있어도 다음 등수를 건너뛰지 않고 촘촘하게(Dense) 채웁니다.
	- 결과: 1등(100), 1등(100), 2등(90), 3등(80)
	- 용도: 공동 1위가 있어도 바로 다음 사람이 2등이 되어야 할 때 씁니다.

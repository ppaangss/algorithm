# 윈도우함수는 SELECT 절 마지막 단계에서 수행된다.

`You cannot use the alias 'RK' of an expression containing a window function in this context.'`

이 에러는 SQL에서 윈도우 함수(Window Function)를 사용한 컬럼의 별칭(Alias)을 동일한 SELECT 절이나 WHERE/GROUP BY/HAVING 절에서 바로 참조하려고 할 때 발생한다.

SQL의 실행 순서 때문에 생기는 문제이다. 

SQL 엔진이 쿼리를 실행할 때, 윈도우 함수($ROW\_NUMBER()$, $RANK()$, $SUM() \text{ OVER}$ 등)의 결과는 SELECT 절의 거의 마지막 단계에서 계산된다. 하지만 `WHERE`나 `GROUP BY`는 그보다 훨씬 먼저 처리된다. 따라서 다른데에서 별칭을 사용하지 못한다.

 가장 깔끔한 해결책은 CTE를 정의해서 별칭을 확정 짓고 바깥쪽에서 사용하면 된다.

[185. Department Top Three Salaries](185.%20Department%20Top%20Three%20Salaries.md) 

```sql
WITH RankedEmployees AS (
    SELECT 
        name, 
        salary, 
        RANK() OVER (ORDER BY salary DESC) AS RK
    FROM employees
)
SELECT * FROM RankedEmployees
WHERE RK <= 10;
```

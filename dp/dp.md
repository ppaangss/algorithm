## 2. 냅색(Knapsack) 보완 사항

냅색 파트에서 **'중복 조합'**과 **'조합'**의 차이를 점화식 순서로 메모해두면 나중에 헷갈리지 않습니다.

- **조합 (0/1 냅색):** 아이템을 한 번만 쓸 때는 무게(j)를 **뒤에서부터(역순으로)** 채워야 합니다. (그래야 '이전 아이템까지의 결과'를 참조함)
    
- **중복 조합 (Unbounded):** 아이템을 무제한 쓸 때는 무게(j)를 **앞에서부터** 채웁니다. (방금 쓴 아이템을 또 쓸 수 있어야 하므로)

## 1. 추가하면 좋은 DP 유형

### **구간 DP (Interval DP)**

작성하신 표 하단에 살짝 언급되어 있는데, 별도 항목으로 빼서 정리하는 것이 좋습니다.

- **특징:** `dp[i][j]`를 **i번째부터 j번째까지의 구간**을 처리하는 데 필요한 최소/최대 비용으로 정의합니다.
    
- **핵심 원리:** 구간을 더 작은 두 개의 구간으로 쪼개어 가며(Divide & Conquer 느낌) 최적해를 구합니다.
    
- **대표 문제:** [파일 합치기 (11066번)](https://www.acmicpc.net/problem/11066), [행렬 곱셈 순서 (11049번)](https://www.acmicpc.net/problem/11049).
    
- **점화식 형태:** `dp[i][j] = min(dp[i][k] + dp[k+1][j] + cost(i, j, k))` (단, `i <= k < j`)
    

### **트리 DP (Dynamic Programming on Trees)**

비트마스킹과 함께 언급하셨지만, 코딩 테스트에서 비트마스킹보다 훨씬 자주 출제됩니다.

- **특징:** 그래프가 트리 구조일 때, 자식 노드의 결과를 바탕으로 부모 노드의 값을 결정합니다.
    
- **핵심 원리:** 주로 DFS(재귀)를 사용하여 **리프 노드부터 루트 노드로** 값을 전파합니다.
    
- **대표 문제:** [트리의 독립집합 (2213번)](https://www.acmicpc.net/problem/2213), [우수 마을 (1949번)](https://www.acmicpc.net/problem/1949).
    

### **전개/상태 전환 DP (State DP)**

단순 선형 DP보다 상태값이 조금 더 복잡한 경우입니다.

- **특징:** `dp[i][state]` 형태. i번째 단계에서 특정 '상태' 정보가 추가로 필요할 때 사용합니다.
    
- **대표 문제:** [RGB거리 (1149번)](https://www.acmicpc.net/problem/1149) - `dp[i][color]` (i번째 집을 빨/초/파로 칠했을 때의 최소 비용).

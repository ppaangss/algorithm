# 냅색 문제: 차원 선택과 최적화 가이드

### 1. 차원 축소의 핵심 원리 (Space Optimization)

보통 냅색은 `dp[i][w]` ($i$ 번째 물건까지 고려, 무게 $w$ 일 때 최대 가치)라는 2차원이 정석입니다. 하지만 '최대 가치'만 궁금하다면 1차원으로 줄일 수 있습니다.

- 어떻게 가능한가?: `dp[i]` 행을 계산할 때 오직 바로 직전인 `dp[i-1]` 행의 정보만 필요하기 때문입니다.
- 핵심 테크닉 (역순 순회): 1차원 배열 하나만 쓰되, 무게를 뒤에서부터(K → w) 채우면 "아직 이번 물건이 반영되지 않은 순수한 이전 상태"를 참조하게 되어 0/1 냅색이 보장됩니다.

근데 이것을 다른 방식으로 생각해볼수 있다. 결국 우리가 구해야하는 것은 최대 가치 V를 구하는 것이다. 따라서 1차원 dp를 무게 W를 인덱스로, 가치 V를 값으로 선택하면 해결할 수 있다.


# 1차원의 한계

하지만 1차원으로 풀 수 없는 경우가 있다.

### 경로나 선택한 물건을 추적해야 할 때

단순히 최대 가치만 구한다면 1차원 배열로 할 수 있지만, 어떤 물건들을 가방에 담았는가? 를 물어본다면 1차원으로 불가능하다.

2차원 dp로 풀 경우 $dp[i][j]$ 에 값이 저장되어 있으므로, 역추적을 통해 $i$ 번째 물건을 넣었을 때 값이 변했는지를 확인할 수 있다. 

그러나 1차원의 경우 확인 자체가 불가능하다.

### 중간 과정의 상태가 독립적으로 필요할 때

문제에서 "아이템을 딱 $K$개 썼을 때의 최댓값"이나 "특정 아이템 그룹 내에서 하나만 골라야 할 때"처럼 특정 행($i$)의 정보가 계산이 끝난 후에도 살아있어야 하는 경우가 있습니다.

- 1차원 배열은 오직 '직전 행'의 정보만 잠시 빌려 쓰는 구조입니다.
- 만약 문제 로직상 $i-2$, $i-3$ 행의 데이터가 동시에 필요하거나, 다른 로직에서 중간 상태의 DP 테이블을 참조해야 한다면 2차원 배열을 유지해야 합니다.


특정 아이템 그룹 내에서 하나만 골라야 할 때
-> 그룹 냅색

**상황 예시:** 당신은 게임 캐릭터의 장비를 맞추고 있습니다.

- **그룹 A (무기):** 검(무게 5), 활(무게 3), 지팡이(무게 2) -> **이 중 딱 하나만** 장착 가능.
    
- **그룹 B (방어구):** 갑옷(무게 10), 로브(무게 4) -> **이 중 딱 하나만** 장착 가능.

### ### "0/1 냅색"이 아닌 다른 변형 문제일 때

냅색 문제 중에는 아이템의 개수 제한이 여러 개인 'Bounded Knapsack'이나, 제약 조건이 무게 하나가 아니라 '무게와 부피' 두 가지인 경우(3차원 DP가 필요함)가 있습니다.

- 조건이 늘어날수록 1차원 최적화는 까다로워지거나 불가능해집니다.
    
- 특히 격자판 DP 문제들(예: RGB거리, 정수 삼각형)은 각 칸의 상태가 명확히 구분되어야 하므로 2차원 구조를 그대로 쓰는 것이 훨씬 안전하고 가독성이 좋습니다.

# 차원 선택

| 상황                | 추천 방식 | 이유                           |
| --------------------- | --------- | -------------------------------- |
| 단순히 최대/최소값만 구할 때  | 1차원   | 메모리 절약 ($O(W)$)                  |
| 선택한 물건 리스트를 출력할 때 | 2차원   | 역추적을 위한 데이터 보존 ($O(N \times W)$) |
| 메모리 제한이 넉넉할 때     | 2차원   | 로직의 가독성이 좋고 실수할 확률이 낮음           |
| 메모리 제한이 매우 빡빡할 때  | 1차원   | $N, W$가 클 때 2차원은 메모리 초과 발생 가능    |


## 1. DP 상태(State)의 구성 요소

보통 DP 테이블을 `dp[i][j]`와 같이 선언할 때, 인덱스인 `i`와 `j`가 바로 상태를 결정하는 요소입니다.

- 결정 요인: 현재 계산 중인 위치(인덱스), 남은 무게(배낭 문제), 현재까지 선택한 개수, 현재의 방향 등.
    
- 값(Value): 해당 상태에서 우리가 구하고자 하는 최적값(최댓값, 최솟값, 경우의 수 등).
    

### 예시: 배낭 문제 (Knapsack Problem)

- 상태 정의: `dp[i][w]`
    
    - `i`: $i$번째 물건까지 고려했을 때
        
    - `w`: 배낭의 남은 용량이 $w$일 때
        
- 의미: "$i$번째 물건까지 살펴봤고 남은 용량이 $w$일 때, 가질 수 있는 최대 가치"
    

---

## 2. 상태를 정의할 때 고려할 점

상태를 잘 정의해야 메모리 초과를 피하고 효율적으로 문제를 풀 수 있습니다.

1. 최소한의 변수: 상태를 표현하는 변수가 많아질수록 차원이 늘어나고(`dp[i][j][k][l]...`), 계산량이 기하급수적으로 증가합니다.
    
2. 독립성: 이전 상태의 결과가 현재 상태를 결정하는 데 충분해야 합니다. (이를 최적 부분 구조라고** 합니다.)
    
3. 중복 제거: 동일한 상태는 반드시 동일한 결과를 반환해야 합니다. (메모이제이션의 핵심)
    

---

## 3. 자주 쓰이는 상태 정의 패턴

| 유형              | 상태 정의 예시     | 설명                                         |
| ------------------- | ---------------- | ---------------------------------------------- |
| 1차원 DP          | `dp[i]`          | $i$번째 위치에서의 최적값 (계단 오르기, 피보나치 등)               |
| 2차원 DP          | `dp[i][j]`       | $i$행 $j$열까지 왔을 때, 혹은 구간 $[i, j]$의 최적값          |
| 상태 압축 (Bitmask) | `dp[i][visited]` | $i$번째 지점에 있고, 방문한 도시 집합이 `visited`일 때 (TSP 문제) |
| 조건부 DP          | `dp[i][0/1]`     | $i$번째 요소를 선택하지 않았을 때(0)와 선택했을 때(1)             |

---

## 4. 상태 전이 (State Transition)

상태를 정의했다면, 한 상태에서 다음 상태로 넘어가는 규칙인 점화식을 세워야 합니다.
****
> **$dp[현재 상태] = f(dp[이전 상태들])$**

예를 들어, 계단 오르기 문제에서 `dp[i]`가 $i$번째 계단까지의 최대 점수라면:

$$dp[i] = \max(dp[i-1], dp[i-2]) + score[i]$$

(단, 문제 조건에 따라 연속 3칸 제한 등이 있다면 상태에 '연속으로 오른 횟수'를 추가해 `dp[i][cnt]` 형태로 확장해야 합니다.)

---

요약하자면:

DP에서 상태는 "문제를 풀기 위해 필요한 최소한의 정보 묶음"입니다. 문제를 마주했을 때 "어떤 정보가 있어야 다음 단계의 답을 구할 수 있을까?"를 고민해보면 상태를 정의하기 수월해집니다.

현재 풀고 계신 특정 문제(예: 백준의 특정 번호)가 있다면 그에 맞춰 더 자세히 설명해 드릴 수 있습니다!

# 조합 (아이템 중심)

동전 한 놈만 팬다 라는 마인드

동전 1원짜리로 모든 금액을 다 채우고, 그 다음에 2원짜리로 다시 모든 금액을 업데이트한다.

2원을 쓸 때는 이미 1원이 다 배치된 상태이니까, 항상 `1, 1, 2` 같은 순서로만 데이터가 쌓인다.

절대로 `2,1,1` 이라는 구조가 나올 수 없다.

```cpp
for (int coin : coins) { // 아이템이 바깥쪽
    for (int j = coin; j <= target; j++) { // 금액이 안쪽
        dp[j] += dp[j - coin];
    }
}
```


# 순열 (금액 중심)

순서가 다른 것도 포함될 때의 경우이다. 즉, 순열

현재 금액에서 쓸 수 있는 모든 카드를 다 꺼낸다는 마인드

1원을 만들 때 1,2,5원을 다 쓰고, 2원을 만들 때, 1,2,5원 다 쓴다.

그럼 결국 이 방식은 3원을 만들 때 `1원 만드는 법 + 2원 추가` 와
`2원 만드는 법 + 1원 추가` 를 모두 카운트 한다.

즉, `{1, 2}`와 `{2, 1}`을 다른 것으로 본다.

```cpp
for (int j = 1; j <= target; j++) { // 금액이 바깥쪽
    for (int coin : coins) { // 아이템이 안쪽
        if (j >= coin) dp[j] += dp[j - coin];
    }
}
```

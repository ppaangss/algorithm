
## 결정적인 차이점

| 구분    | BFS (너비 우선 탐색)          | DP (다이나믹 프로그래밍)          |
| ----- | ----------------------- | ------------------------ |
| 핵심 목적 | 최단 경로/최소 횟수 찾기          | 최적의 값(최대/최소) 또는 경우의 수 찾기 |
| 가중치   | 모든 간선의 가중치가 동일(보통 1)할 때 | 가중치가 다양하거나 점수가 누적될 때     |
| 상태 관계 | 상하좌우 이동 등 공간적 연결        | 이전 단계의 결과를 이용하는 논리적 연결   |
## 2. 언제 무엇을 쓰는가? (판단 기준)

### **💡 BFS를 써야 할 때**

- **"최소 몇 번 만에 도착하는가?"**를 물을 때.
    
- 가중치가 없는 그래프나 격자판에서 이동할 때.
    
- **방문 체크(`visited`)**가 중요하고, 한 번 간 곳은 다시 안 가도 될 때.
    
- **예시:** 미로 찾기, 숨바꼭질, 네트워크 연결 요소 개수.
    

### **💡 DP를 써야 할 때**

- **"얻을 수 있는 최대 점수/최소 비용은?"**을 물을 때.
    
- **"방법의 가짓수(경우의 수)"**를 모두 구해야 할 때.
    
- 이미 계산한 값을 또 써야 하는 **중복 계산**이 엄청나게 많을 때.
    
- 순서가 정해져 있어 뒤로 돌아갈 일이 없을 때 (방향성).
    
- **예시:** 계단 오르기, 배낭 채우기(Knapsack), RGB거리, LIS/LCS.

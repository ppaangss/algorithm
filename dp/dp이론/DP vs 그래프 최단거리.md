
### 공통점: "최적 부분 구조 (Optimal Substructure)"

두 방식 모두 다음의 논리를 따릅니다.

- 논리: "지점 C까지 가는 최단 거리는, (시작점 ~ B까지의 최단 거리) + (B ~ C까지의 거리)와 같다."
    
- 즉, 큰 문제(A→C)를 해결하기 위해 작은 문제(A→B)의 최적 해를 미리 구해놓고 재사용한다는 점이 완전히 동일합니다.

### 결정적인 차이: "순서(Order)"와 "순환(Cycle)"

가장 큰 차이는 계산하는 순서를 정할 수 있느냐 없느냐에 있습니다.

#### DP (Dynamic Programming)

- 순서가 명확함: 계단 오르기처럼 $i-1, i-2$를 계산해야만 $i$를 계산할 수 있는 명확한 방향성이 있습니다. (이를 DAG: 방향성 비순환 그래프라고 합니다.)
    
- 특징: 뒤로 돌아가는 일이 절대 없습니다. 따라서 한 번 계산한 $dp[i]$는 절대 바뀌지 않습니다.
    

#### 그래프 최단 거리 (예: 다익스트라)

- 순서가 불분명함: 어느 노드를 먼저 방문해야 할지 미리 알기 어렵습니다. 그래서 '우선순위 큐'를 써서 현재 가장 가까운 곳부터 탐색합니다.
    
- 순환(Cycle) 가능성: 그래프는 길을 가다 보면 다시 원래 자리로 돌아올 수도 있습니다. BFS나 다익스트라는 이런 순환을 `visited`나 `dist` 배열 갱신으로 해결합니다.


| 구분    | DP (다이나믹 프로그래밍)   | 그래프 최단 거리 (다익스트라/BFS)    |
| --------- | --------------------- | ---------------------------- |
| 상태 전이 | 점화식에 의해 순서대로 진행       | 인접한 노드를 탐색하며 진행              |
| 구조    | 계층이 확실함 (1층→2층→3층)    | 관계가 얽혀 있음 (A-B-C-A)          |
| 값 갱신  | 한 번 확정된 $dp[i]$는 고정   | 더 짧은 경로를 찾으면 `dist[i]` 갱신 가능 |
| 예시    | 카운트다운: 점수가 깎이기만 함 | 미로 찾기: 왔던 길 근처로 돌아갈 수 있음 |

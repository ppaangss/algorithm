
# Top-Down (재귀) vs Bottom-Up (반복문)

- **Top-Down**: "10원을 만들려면 9원 만드는 법과 8원 만드는 법을 알아야 해!" 하며 큰 문제에서 작은 문제로 내려갑니다. (재귀함수 + 메모이제이션)
    
- **Bottom-Up**: "0원부터 시작해서 10원까지 차근차근 쌓아 올리자!" (반복문). 보통 백준 2293번처럼 메모리 제한이 빡빡하면 Bottom-Up이 유리합니다.


# 냅색(Knapsack) 알고리즘 (중복 허용 여부)

이번 '동전 1' 문제처럼 **동전을 무한히 쓸 수 있는가**, 아니면 **딱 한 번씩만 쓸 수 있는가**의 차이입니다.

- **무한 사용**: 금액을 `coin`부터 `target`까지 **앞으로** 가며 채웁니다. (현재 동전을 또 쓸 수 있으니까요.)
    
- **한 번만 사용**: 금액을 `target`부터 `coin`까지 **거꾸로** 채웁니다. (뒤에서부터 채워야 현재 단계에서 넣은 동전이 다음 계산에 영향을 주지 않기 때문입니다.)


# 상태의 정의 (차원 늘리기)

가끔은 `dp[금액]` 1차원만으로는 부족할 때가 있습니다.

- 예: "동전을 연속해서 3개 이상 쓰면 안 된다"는 조건이 붙는다면?
    
- `dp[금액][연속사용횟수]`처럼 **2차원 배열**로 상태를 확장해서 생각해야 합니다. 이를 "상태 DP"라고 부릅니다.

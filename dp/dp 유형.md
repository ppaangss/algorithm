
# 선형 DP

가장 기본이 되는 유형으로, 앞에서부터 차례대로 선택하며 최적의 값을 쌓아 나가는 방식입니다.

- 특징: `dp[i]`가 `dp[i-1]`, `dp[i-2]` 등 바로 직전의 값들에만 의존함.
- 대표 문제:
    - 2×n 타일링 (2133번 등): $dp[n] = dp[n-1] + dp[n-2]$ (피보나치 형태)
    - 계단 오르기 (2579번): 마지막 계단을 밟을 때 '한 칸 전' 혹은 '두 칸 전'에서 오는 경우를 나눔.
    - 1, 2, 3 더하기 (9095번): 금액 중심의 순열 DP. $dp[n] = dp[n-1] + dp[n-2] + dp[n-3]$
- 핵심: "마지막 단계에서 내가 할 수 있는 선택지가 무엇인가?"를 고민하면 점화식이 나옵니다.


# 냅색 유형 (조합/자원 배분)

제한된 가방(또는 금액) 안에 아이템을 효율적으로 넣는 유형

아이템의 종류와 무게 라는 두 가지 변수가 존재한다.

- 유형
	- 0/1 냅색
		- 아이템을 딱 한 번씩만 사용할 수 있음. 
	- Unbounded 냅색
		- 아이템을 무제한으로 사용할 수 있음
	- 순열
		- 아이템을 구성하는 순서가 중요함
	- 조합
		- 아이템을 구성하는 순서는 중요하지 않고 어떤 것을 선택했는지 중요함.

# 격자판 DP

지도 위에서 최단 경로를 찾거나 점수를 쌓는 유형

특징: `dp[i][j]`는 보통 `(i, j)` 위치까지 도달했을 때의 최댓값/경우의 수를 의미함.

핵심: 위(`i-1`)나 왼쪽(`j-1`)에서 오는 경우 중 최적의 값을 선택합니다.

# 4. 연속/부분 수열 DP (LIS / LCS)

문자열이나 숫자 배열의 일부분을 다루는 유형입니다.

- 특징: 두 문자열을 비교하거나, 배열 내에서 특정 조건을 만족하는 부분 수열을 찾음.
    
- 대표 문제:
    
    - 가장 긴 증가하는 부분 수열 (LIS - 11053번): 자신보다 작은 앞쪽 원소들의 DP 값 중 최대를 찾음.
    - 가장 긴 공통 부분 수열 (LCS - 9251번): 두 문자열의 문자가 같을 때와 다를 때로 나눔.

# 5. 비트마스킹 / 트리 DP (심화)

상태가 아주 많거나 계층 구조일 때 사용합니다. 초보 단계 이후에 정복하게 될 영역입니다.

- **비트마스킹**: 방문한 도시들을 이진수(비트)로 표현하여 상태를 저장 (예: 외판원 순회 문제).
- **트리 DP**: 부모 노드의 값이 서브트리(자식들)의 값에 의존하는 경우.


|**유형**|**핵심 질문 (이걸 스스로 물어보세요!)**|**대표적인 문제 예시**|
|---|---|---|
|**1. 선형 DP**|"마지막 칸에 오기 직전, **한 칸 전**이었나 **두 칸 전**이었나?"|1,2,3 더하기, 계단 오르기, 타일 채우기|
|**2. 냅색(Knapsack)**|"이 물건을 **넣을까 말까?** 넣는다면 **무게**가 얼마나 남지?"|동전 1, 평범한 배낭|
|**3. 격자판 DP**|"이 좌표로 오려면 **위**에서 왔나, **왼쪽**에서 왔나?"|정수 삼각형, 등굣길, RGB거리|
|**4. 수열(LIS/LCS)**|"현재 숫자보다 **작은 숫자들** 중에 가장 긴 줄이 어디지?"|가장 긴 증가하는 부분 수열(LIS)|
|**5. 상태/구간 DP**|"이 **범위(i~j)**를 해결하려면 어디를 기준으로 쪼개야 하지?"|행렬 곱셈 순서, 파일 합치기|


# 단계

- **유형 분류**: "이건 위 5가지 중 어디에 속할까?" (예: 냅색이다!)
    
- **점화식 세우기**: "마지막 상태를 만들기 위한 직전 상태는?" (예: `dp[i] = dp[i-coin] + ...`)
    
- **손으로 3단계만 해보기**: `n=1, 2, 3`일 때 내 점화식이 맞는지 검증하기.

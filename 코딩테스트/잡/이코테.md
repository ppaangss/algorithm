**계수정렬**
특정한 조건이 부합할 때만 사용할 수 있지만 매우 빠른 정렬 알고리즘
데이터의 개수가 N, 데이터 중 최댓값이 K 일 때, 계수 정렬의 시간복잡도는 $O(N + K)$ 이다.
다만, 계수 정렬은 데이터의 크기 범위가 제한 되어 정수 형태로 표현할 수 있을 때만 사용 가능하다.
예를들어 성적 데이터를 정렬할 때 계수 정렬이 효과적이다.
성적은 0부터 100까지의 정수로 되어있다고 가정하면 배열을 `int num[101]` 을 만든다.
그리고 성적 데이터를 한 번 순회한다. 해당 성적에 해당하는 인덱스에 +1을 해준다.
결과적으로 배열의 값은 해당 성적을 맞은 사람의 수일 것이다.
그런뒤, 해당 배열의 값 만큼 인덱스를 출력해준다. 그러면 정렬이 된다.

계수정렬은 시간복잡도가 매우 빠르므로, 잘 쓰면 좋다.
하지만 매번 가능한 것이 아니다. 계수 정렬은 때에 따라 심각한 비효율성을 초래할 수 있다.
예를들어 데이터가 100만개가 가능한 상황에서 값이 0과 999999, 2개만 존재한다고 가정할 때, 이때의 경우에도 배열을 똑같이 100만 개가 되도록 설정해야 하므로, 공간 낭비가 심하다.
대신, 동일한 값을 가지는 데이터가 여러 개 등장할 때 유리하다.
따라서 계수정렬은 일반 정렬 알고리즘인 $O(NlogN)$ 이 안될 때 계수정렬을 사용하기에 유리한 조건이 나온다면 생각해보면 좋을 것 같다.

**결정 알고리즘**
최적화 문제를 결정 문제로 바꾸는 파라메트릭 서치
원하는 조건을 만족하는 가장 알맞은 값을 찾는 문제
범위 내에서 조건을 만족하는 가장 큰 값을 찾으라는 최적화 문제라면 이진 탐색으로 결정 문제를 해결하면서 범위를 좁혀갈 수 있다.

결국은 결정 알고리즘이라고 판단할 수 있는 근거는 
1. 범위 내에서 조건을 찾는 문제 라는 것을 인지해야한다.
2. log N 의 시간복잡도를 가진 알고리즘을 써야할 것 같다.

이 2가지일 것같다.

**dp**
재귀의 시간복잡도는 일반적으로 2번 들어가면 (피보나치) $O(2^N)$ 이다.

재귀함수를 이용하여 DP를 구현하는 것을 탑다운 방식
반복문을 이용하여 작은 문제부터 구하는 것을 바텀업 방식

어떤 문제를 풀 때 완전탐색으로 안풀린다면 dp를 의심하자.
시간복잡도를 줄일 수 있는 방법이다.

**다익스트라**

우선순위 큐를 쓰는 다익스트라는 $O(ElogV)$ 로 풀 수 있다. (V는 노드개수, E는 간선수)

**플로이드 워셜**
모든 지점에서 다른 모든 지점까지의 최단 경로를 모두 구해야 하는 경우 사용하는 알고리즘
노드 개수가 적은 경우에 유리하며, 따라서 인접행렬을 쓰는게 유리
총 시간 복잡도는 $O(N^3)$ 이다.

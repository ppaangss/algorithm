
# 2차원 벡터에 요소 넣기, 2차원 벡터 초기화

**미리 크기가 정해져있는 벡터 만들기** 

```cpp
vector<vector<int>> v(N,vector<int>(M,0));

for (int i = 0; i < N; i++) {  
    for (int j = 0; j < M; j++) {
        cin >> v[i][j];
    }
}
```


이딴건 안됨

```cpp
vector<vector<int>> v;

for (int i = 0; i < N; i++) {  
    for (int j = 0; j < M; j++) {
        int tmp; cin >> tmp;
        v[i].push_back(tmp);
    }
}
```

이렇게 해야함

```cpp

vector<vector<int>> v;

for (int i = 0; i < N; i++) {  
    v.push_back(vector<int>());
    for (int j = 0; j < M; j++) {
        int tmp; cin >> tmp;
        v[i].push_back(tmp);
    }
}
```

또는 이렇게 하기

```cpp
vector<vector<int>> v;

for (int i = 0; i < N; i++) {
    vector<int> row; // 한 행을 담을 임시 벡터
    for (int j = 0; j < M; j++) {
        int tmp; cin >> tmp;
        row.push_back(tmp);
    }
    v.push_back(row); // 완성된 행을 통째로 v에 추가
}
```

# 2차원 벡터를 전역변수로 사용

assign() 함수를 이용하면 간단하게 해결 가능하다.

```cpp
#include <vector>

using namespace std;

int N; int M;
vector<vector<int>> v;

int main(){
	cin >> N >> M;
	// 5행 10열로 리사이즈하며 모든 값을 -1로 초기화
	v.assign(N, vector<int>(M, -1));
}
```

# 2차원 벡터를 다시 쓰자

예를들어 BFS를 여러번 수행해야 한다고 가정하다. 그러면 하나의 visited 벡터를 여러번 사용해야 할 때가 있다. 그럴 경우 초기화 시켜주는 방법은 뭐가 적당할까? 

- 2차원 벡터이면 assign
- 2차원 배열이면 fill

## assign()

이미 선언되어 있는 2차원 벡터의 크기를 값을 덮어씌울 때 사용한다.

만약 크기가 다른 벡터를 다시 만든다면 위에 2차원 벡터를 전역변수를 사용할 때 처럼 한 번의 assign을 해주는 것이 더 좋다.

```cpp
vector<vector<int>> v;

// 5행 10열로 리사이즈하며 모든 값을 -1로 초기화
for (auto& row : v) {
    row.assign(row.size(), -1); // 기존 메모리 공간에 값만 덮어씌움
}
```

## fill()

반복문을 써서 일일이 `fill()` 로 채울 수 있다.

일반적으로 벡터 보다는 배열일 때 사용하면 매우좋다. 

```cpp
#include <algorithm>

bool visited[100][100];

int main(){
	int N; cin >> N;
	int M; cin >> M;
}
// 초기화할 때 (N행 M열 기준)
std::fill(&visited[0][0], &visited[0][0] + sizeof(visited), false);

// 또는 더 안전하게 (행*열 개수만큼)
std::fill(&visited[0][0], &visited[0][0] + (N * M), false);
```

## memset

memset은 절대 쓰면 안된다!!!!!

memset은 연속되어있는 메모리를 지우는 기능인데 벡터의 경우 노드들이 메모리가 연속적으로 이루어져 있지 않는다. 따라서 배열에서는 가능한데 벡터에서는 사용하면 안된다.

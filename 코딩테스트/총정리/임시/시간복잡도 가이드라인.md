
# 시간복잡도 기준

| 입력 크기 (N)                            | 허용 시간 복잡도           | 대표 알고리즘                         |
| ------------------------------------ | ------------------- | ------------------------------- |
| $N \le 10$                           | $O(N!)$, $O(3^N)$   | 순열, 모든 경우의 수 탐색                 |
| $N \le 20 \sim 25$                   | $O(2^N)$            | 브루트포스(전수 조사), 비트마스킹             |
| $N \le 100$                          | $O(N^4)$            | 4중 for문, 복잡한 DP                 |
| $N \le 200 \sim 500$                 | $O(N^3)$            | 플로이드-워셜(최단 경로), 3중 for문         |
| $N \le 2,000 \sim 5,000$             | $O(N^2)$            | 단순 DP, 2중 for문, 선택/삽입 정렬        |
| **$N \le 1,000,000 \sim 5,000,000$** | $O(N \log N)$       | 정렬(Sort), 우선순위 큐, 세그먼트 트리, 이분탐색 |
| $N \le 10,000,000 \sim 20,000,000$   | $O(N)$              | 단순 반복문, 슬라이딩 윈도우, 투 포인터         |
| $N > 10^8$ (그 이상)                    | $O(\log N)$, $O(1)$ | 이진 탐색(Binary Search), 수학적 공식    |

# $O(\sqrt{N})$ vs $O(log{N})$

$O(\sqrt{N})$ 와 $O(log{N})$ 의 시간복잡도는 비슷해 보이지만 비슷하지 않다.

예를들어 $N = 1000000$ 이라고 가정하면 대략적인 시간복잡도는 다음과 같다.

| 복잡도           | 계산식             | 실제 연산 횟수 (약) |
| ----------------- | ------------------- | ---------------- |
| $O(\log N)$   | $\log_2(1,000,000)$ | 약 20번        |
| $O(\sqrt{N})$ | $\sqrt{1,000,000}$  | 1,000번       |

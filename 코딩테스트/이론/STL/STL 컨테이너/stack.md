# 개요
- 가장 나중에 들어온 데이터가 가장 먼저 나가는 자료구조이다.
- 데이터의 삽입과 삭제가 한쪽 끝(Top)에서만 일어난다.
- 중간에 있는 원소에 직접 접근하거나 수정할 수 없다.
- 재귀 알고리즘, DFS(깊이 우선 탐색), 실행 취소(Undo) 기능 등에 주로 사용된다.

# 설명
스택은 마치 '쌓여 있는 접시'와 같습니다. 새로운 접시를 놓을 때도 맨 위에 놓아야 하고, 접시를 꺼낼 때도 맨 위의 접시부터 꺼내야 하는 원리입니다. 삽입과 삭제가 항상 한 방향에서만 이루어지기 때문에 데이터의 순서를 역전시켜야 할 때 매우 유용합니다.

## 특징
- 단방향 접근: 데이터의 추가/삭제가 `top`이라고 불리는 한 지점에서만 발생한다.
- 제한적 접근: 인덱스를 통한 중간 원소 탐색이나 접근이 불가능하다. (오직 `top`만 확인 가능)
- 효율성: 삽입과 삭제가 항상 끝단에서만 일어나므로 속도가 매우 빠르다.
- 활용도: 함수 호출의 콜 스택(Call Stack), 수식 계산(후위 표기법), 문자열 뒤집기 등에서 핵심적으로 사용된다.

## 자주 쓰는 멤버 함수

|함수|설명|시간 복잡도|
|---|---|---|
|push(Value)|스택의 맨 위에 원소를 추가함|$O(1)$|
|pop()|스택의 맨 위(가장 최근 추가된) 원소를 제거함|$O(1)$|
|top()|스택의 맨 위 원소를 반환 (제거하지 않음)|$O(1)$|
|empty()|스택이 비어있으면 `true`, 아니면 `false` 반환|$O(1)$|
|size()|현재 스택에 들어있는 원소의 개수를 반환|$O(1)$|

# 코드 예시

```cpp
#include <iostream>
#include <stack> // 스택 헤더 포함

using namespace std;

int main() {
    // 1. 스택 선언
    stack<int> s;

    // 2. 데이터 삽입 (push)
    s.push(10);
    s.push(20);
    s.push(30);

    // 3. 현재 top 확인 및 크기 출력
    cout << "현재 top: " << s.top() << endl;   // 출력: 30
    cout << "스택 크기: " << s.size() << endl; // 출력: 3

    // 4. 데이터 삭제 (pop) 및 순차 출력
    cout << "스택 원소 출력: ";
    while (!s.empty()) {
        cout << s.top() << " "; // 맨 위의 원소 확인
        s.pop();                // 맨 위의 원소 제거
    }
    // 출력 결과: 30 20 10 (넣은 순서의 역순)

    cout << "\n삭제 후 비어있는지 확인: " << (s.empty() ? "Yes" : "No") << endl;

    return 0;
}
```

# 활용
- DFS (깊이 우선 탐색): 그래프 탐색 시 경로를 저장하는 용도.
- 괄호 검사: `(`, `[` 등의 괄호 쌍이 맞는지 확인할 때 유용.
- 역순 정렬: 들어온 순서를 반대로 뒤집어야 할 때.

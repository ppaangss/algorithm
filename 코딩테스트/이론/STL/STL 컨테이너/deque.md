
# 1. 데크의 3대 특징 

- **양방향 $O(1)$**: 맨 앞(`push_front`, `pop_front`)과 맨 뒤(`push_back`, `pop_back`) 삽입/삭제가 모두 매우 빠릅니다.
- **인덱스 접근 가능**: `dq[i]`로 중간 원소에 바로 접근할 수 있습니다 ($O(1)$).
- **정렬 가능**: `std::sort(dq.begin(), dq.end())`를 사용할 수 있습니다.


|**분류**|**함수**|**설명**|
|---|---|---|
|**삽입**|`push_back(x)`, `push_front(x)`|뒤/앞에 원소 추가|
|**삭제**|`pop_back()`, `pop_front()`|뒤/앞에서 원소 제거|
|**참조**|`dq[i]`, `dq.front()`, `dq.back()`|인덱스/맨 앞/맨 뒤 원소 확인|
|**크기**|`dq.size()`, `dq.empty()`|현재 개수 확인 / 비었는지 체크|
|**기타**|`dq.clear()`, `dq.begin()`, `dq.end()`|전체 삭제 / 반복자 반환|

#### ✅ 1) 큐(`queue`)를 쓰는데 중간 값을 확인해야 할 때

일반 큐는 `q[i]`가 안 되지만, 데크는 됩니다. 큐의 기능이 필요하면서 "안에 뭐가 들었지?" 하고 순회하거나 특정 인덱스를 봐야 한다면 무조건 데크입니다.

#### ✅ 2) 슬라이딩 윈도우 (최댓값/최솟값 찾기)

특정 구간을 밀면서 앞을 빼고 뒤를 넣어야 할 때 가장 효율적입니다. 특히 **"덱을 이용한 최댓값 찾기"** 알고리즘은 단골 심화 문제입니다.

#### ✅ 3) 앞뒤 반전이 잦은 문제 (백준 5430번 'AC' 스타일)

문자열이나 배열을 뒤집으라는 명령이 올 때, 실제로 뒤집으면 $O(N)$이라 시간 초과가 납니다. 이때 데크를 써서 "지금은 앞이 입구야", "지금은 뒤가 입구야"라고 **방향 상태(flag)**만 바꿔주면 $O(1)$로 처리할 수 있습니다.


```cpp
#include <iostream>
#include <deque>
#include <algorithm>

using namespace std;

int main() {
    deque<int> dq;

    dq.push_back(10);  // [10]
    dq.push_front(20); // [20, 10]
    dq.push_back(30);  // [20, 10, 30]

    // 인덱스 접근
    cout << dq[1] << endl; // 10 출력

    // 정렬 (vector처럼 사용 가능)
    sort(dq.begin(), dq.end()); // [10, 20, 30]

    // 앞뒤 삭제
    dq.pop_front(); // [20, 30]
    dq.pop_back();  // [20]

    return 0;
}
```

# 개요
- 중복을 허용하지 않는 자료구조이다.
- 데이터를 항상 정렬된 상태로 유지한다.
- 원소의 값 자체를 수정할 수 없다.
- 특정 원소가 있는지 빠르게 검색해야 할 때 사용할 수 도 있다.

# 설명
set 자료구조는 중복된 값을 저장하지 못하는 자료구조이다. 

따라서, 이미 존재하는 값을 삽입할 경우 무시된다. 

## 특징
- 내부적으로 레드-블랙 트리(Red-Black Tree)라는 균형 이진 탐색 트리 구조를 사용한다.
	- 한쪽으로 치우치지 않게 트리의 높이를 일정하게 유지해, 데이터가 많아져도 탐색 속도가 느려지지 않아, 검색 속도가 매우 빠르다.
	- 자동 정렬로 인해 새로운 원소가 들어올 때 마다 트리의 노드들과 크기를 비교해 적절한 위치에 배치된다. 따라서 자동으로 오름차순(기본값)으로 정렬된다.
- 원소의 값 자체를 직접 수정할 수 없다. 값을 바꾸려면 기존 원소를 삭제하고 새 원소를 삽입해야 한다.

## 시간 복잡도

| 작업          | 시간 복잡도      | 설명               |
| ----------- | ----------- | ---------------- |
| 삽입 (Insert) | $O(\log n)$ | 위치를 찾아 트리 균형을 맞춤 |
| 삭제 (Erase)  | $O(\log n)$ | 원소 삭제 후 트리 재정렬   |
| 탐색 (Find)   | $O(\log n)$ | 이진 탐색과 유사한 속도    |

# 예시
## 코드 예시

```java
#include <iostream>
#include <set>

using namespace std;

int main() {
    set<int> s;

    // 1. 삽입
    s.insert(30);
    s.insert(10);
    s.insert(20);
    s.insert(10); // 중복 삽입 시도 (무시됨)

    // 2. 출력 (자동 정렬되어 있음: 10 20 30)
    for (int n : s) {
        std::cout << n << " ";
    }

    // 3. 탐색
    if (s.find(20) != s.end()) {
        cout << "\n20이 존재합니다.";
    }

    // 4. 삭제
    s.erase(20);

    return 0;
}

```

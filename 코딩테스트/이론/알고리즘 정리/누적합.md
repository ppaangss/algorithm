
배열의 특정 구간의 합을 아주 빠르게 구하기 위해 사용하는 알고리즘
매번 구간 합을 구하기 보다, 미리 합계를 계산해 둔 배열을 하나 더 만들어 놓는 것이 핵심이다.

## 원리

수열 A가 있을 때, 누적 합 배열 S의 각 원소 $S[i]$ 는 수열의 첫 번째 원소부터 $i$ 번째 원소까지의 합을 의미한다.

- $S[0] = A[0]$
- $S[1] = A[0] + A[1]$
- $S[2] = A[0] + A[1] + A[2]$
- ...
- $S[i] = S[i-1] + A[i]$

$i$번째부터 $j$번째까지의 합을 알고 싶다면, 전체 합($j$까지의 합)에서 앞부분($i-1$까지의 합)을 빼면 된다.
$구간 합(i \sim j) = S[j] - S[i-1]$

## 특징

- 시간복잡도
	- 만약 배열의 크기가 N이고 M번 정도 구간 합을 구해야 할 경우
	- 일반적인 방법으로 매번 i부터 j까지 더하면 시간복잡도는  $O(N \times M)$ 이다.
	- 하지만 누적합을 사용하면 처음에 누적 합 배열을 만드는데 $O(N)$ 이 걸리지만, 이후 구간 합을 구할 때 단 한번의 뺄셈으로 끝나므로 총 M번 하면 $O(M)$ 이 걸려 최종적으로 시간복잡도가 $O(N + M)$ 이 걸리게 된다.
	- 누적 합을 사용하므로써 시간복잡도를 줄일 수 있다.


## 구현 예시

```c++
int a[100004], psum[100004]; 
for(int i = 1; i <= n; i++) { 
	cin >> a[i]; 
	// i번째 누적합 = (i-1)번째까지의 누적합 + 현재 값 
	psum[i] = psum[i-1] + a[i]; 
}
```

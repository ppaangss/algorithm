
모든 정점 사이의 최단 경로를 한 번에 구하는 알고리즘

다익스트라가 하나의 시작점으로부터 다른 모든 정점까지의 거리를 구하는 것과 달리,
플로이드-워셜은 모든 노드 쌍에 대한 최단 거리를 구한다는 점이 가장 큰 특징이다.

## 원리

점 i 에서 점 j 로 바로 가는 것 보다, 어떤 점 k를 거쳐서 가는 것이 더 빠를까?

$$dist[i][j] = \min(dist[i][j], dist[i][k] + dist[k][j])$$

- $dist[i][j]$ : $i$에서 $j$로 가는 현재까지의 최단 거리
- $dist[i][k] + dist[k][j]$ : $i$에서 $k$를 거쳐 $j$로 가는 거리

### 동작 과정

1. 초기화
	- 모든 정점 쌍의 거리를 저장하는 2차원 배열을 만든다.
	- 연결된 간선이 있으면 그 가중치를 적고, 없으면 `무한대(INF)`로 채웁니다.
2. 3중 for문 수행
	- 1번 노드를 거쳐가는 경우를 고려하여 모든 ( i,j ) 거리를 갱신합니다.
	- 2단계: 2번 노드를 거쳐가는 경우를 고려하여 다시 갱신합니다.
	- n단계: 마지막 $n$번 노드까지 모두 거쳐가는 경우를 확인합니다.
3. 최종적으로 남은 배열의 값이 각 쌍의 최단 거리가 된다.

## 특징

- 시간복잡도
	- $O(V^3)$ ($V$는 정점의 개수)
- 공간복잡도
	- $O(V^2)$ (2차원 배열)
- 음수 가중치 가능하다.

## 구현 예시

3중 for 문

```c++
for (int k = 1; k <= n; k++) { // k: 거쳐가는 노드 
	for (int i = 1; i <= n; i++) { // i: 출발 노드 
		for (int j = 1; j <= n; j++) { // j: 도착 노드 
			if (dist[i][k] + dist[k][j] < dist[i][j]) { 
				dist[i][j] = dist[i][k] + dist[k][j]; 
			} 
		} } }
```

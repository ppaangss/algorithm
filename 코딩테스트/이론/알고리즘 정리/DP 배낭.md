
DP 배낭 문제(Knapsack Problem)는 다이나믹 프로그래밍의 가장 대표적인 고전 예제 중 하나입니다. 한정된 용량의 배낭에 최대의 가치를 갖도록 물건을 넣는 최적화 문제를 말합니다.

## 1. 문제의 정의

- **주어진 것**: 배낭의 최대 수용 무게 $W$, 각 물건의 무게 $w_i$와 가치 $v_i$.
- **목표**: 배낭에 담은 물건들의 무게 합이 $W$를 초과하지 않으면서, 가치의 합을 **최대화**.
- **핵심 제약**: 각 물건은 통째로 넣거나, 아예 넣지 않아야 합니다 ($0$ 또는 $1$).

---

## 2. 상태 정의 및 점화식

냅색 문제를 풀기 위해서는 두 가지 변수를 축으로 하는 2차원 DP 테이블이 필요합니다.

### 상태 정의

`dp[i][j]` = **처음부터 $i$번째 물건까지 고려했을 때, 배낭의 임시 용량이 $j$인 경우의 최대 가치**

### 점화식 (State Transition)

$i$번째 물건을 검사할 때, 우리는 두 가지 선택을 할 수 있습니다.

1. **물건 $i$를 넣지 않는 경우**: 이전 물건까지 고려했을 때의 가치를 그대로 가져옵니다.
    
    - `dp[i][j] = dp[i-1][j]`
        
2. **물건 $i$를 넣는 경우**: (현재 용량 $j$ $\ge$ 물건 무게 $w_i$일 때만 가능)
    
    - 물건 $i$의 가치 + (물건 $i$를 넣기 전 남은 용량 $j - w_i$에서의 최대 가치)
        
    - `dp[i][j] = v_i + dp[i-1][j - w_i]`
        

최종 점화식은 이 두 값 중 **최대값**을 선택하는 것입니다.

$$dp[i][j] = \max(dp[i-1][j], v_i + dp[i-1][j - w_i])$$

## 3. 동작 과정 예시

예: 배낭 용량 **7kg**, 물건 목록: (6kg, 13만원), (4kg, 8만원), (3kg, 6만원), (5kg, 12만원)

|**물건(무게, 가치)**|**1kg**|**2kg**|**3kg**|**4kg**|**5kg**|**6kg**|**7kg**|
|---|---|---|---|---|---|---|---|
|**(6, 13)**|0|0|0|0|0|13|13|
|**(4, 8)**|0|0|0|8|8|13|13|
|**(3, 6)**|0|0|6|8|8|13|**14**|

_마지막 칸의 14는 (4kg, 8만원)과 (3kg, 6만원)을 조합하여 만든 최적해입니다._

## 4. 공간 최적화 (1차원 DP)

2차원 배열은 직관적이지만, 사실 `dp[i]` 행을 계산할 때 오직 `dp[i-1]` 행의 정보만 필요합니다. 따라서 **1차원 배열** 하나로 메모리를 절약할 수 있습니다.

**주의점**: 1차원 배열 사용 시 용량($j$)을 **뒤에서부터(역순으로)** 갱신해야 합니다. 앞에서부터 갱신하면 같은 물건이 중복해서 담기는 '무한 배낭 문제(Unbounded Knapsack)'가 되어버리기 때문입니다.


```c++
// 1차원 DP 냅색 예시
for (int i = 0; i < n; i++) { // 각 물건에 대해
    for (int j = W; j >= weight[i]; j--) { // 배낭 용량을 뒤에서부터 훑음
        dp[j] = max(dp[j], dp[j - weight[i]] + value[i]);
    }
}
```

## 5. 배낭 문제의 종류

- **0/1 Knapsack**: 물건을 쪼갤 수 없음 (DP로 해결).
- **Fractional Knapsack**: 물건을 설탕이나 금가루처럼 쪼갤 수 있음 (그리디 알고리즘으로 해결).
- **Unbounded Knapsack**: 각 물건의 개수가 무한함 (DP 상에서 용량을 정순으로 순회).

이 개념이 아까 질문하신 '완전범죄' 문제에 적용된 것입니다. 그 문제에서는 'B도둑의 흔적'이 배낭의 무게가 되고, 'A도둑의 흔적'이 최소화해야 할 비용이 된 것이죠.

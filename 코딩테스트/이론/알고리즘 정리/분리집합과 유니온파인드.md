# 유니온 파인드 (Union-Find)

공통 원소가 없는 상호 배타적인 부분 집합들로 나눠진 원소들에 대한 정보를 저장하고 조작하는 자료구조 

유니온 파인드의 3가지 연산

1. 초기화
	- 모든 노드의 루트노드는 자기자신으로 시작한다.
2. 합치기 (Union): 
	- 두 노드에 대한 루트노드를 비교하여 다른 경우는 다른 집합이므로 서로 합쳐준다.
3. 찾기 (Find): 
	- 해당 집합의 루트 노드를 찾는다. 이 루트노드가 같은 노드는 하나의 집합에 속한 것이다.

# 최적화

### 1. 경로 압축 (Path Compression)

- **핵심:** `find` 연산을 수행할 때마다 만나는 모든 노드의 부모를 **최상위 루트**로 바로 연결합니다.
- **효과:** 처음 한 번은 루트를 찾으러 올라가야 하지만, 그 이후부터는 질문하신 대로 $O(1)$에 가까운 속도로 루트를 찾을 수 있습니다.
- **비유:** 조부모님, 증조부모님을 거쳐 가던 족보를 정리해서, 모두가 가문의 수장(루트)과 직접 연락할 수 있게 '단축 번호'를 저장하는 것과 같습니다.

### 2. Union by Rank

- **핵심:** 각 집합의 '높이(Rank)'를 기억해 두었다가, **높이가 낮은 트리를 높은 트리 아래에** 붙입니다.
- **효과:** 트리가 한쪽으로 길게 늘어지는(Skewed) 현상을 막아줍니다. 경로 압축을 하지 않더라도 트리의 높이를 $O(\log N)$으로 유지해 줍니다.
- **참고:** 실전에서는 Rank 대신 집합의 원소 개수(Size)를 기준으로 하는 **Union by Size**를 쓰기도 합니다. 둘 다 목적은 "트리의 균형을 맞추는 것"으로 동일합니다.

# 시간복잡도

일반적으로 $O(\alpha(N))$ 라고 애커만 역함수라고 불리는 시간만큼 걸린다는데 사실상 $O(1)$ 이라고 봐도 무방하다.

# 구현

### 경로압축 + Union by Rank

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

// 최대 노드 수에 맞춰 배열 크기 조절
vector<int> parent;
vector<int> rank_node;

// 1. Find 연산: 경로 압축(Path Compression)
// 재귀를 돌며 만나는 모든 노드의 부모를 루트로 직접 연결함
int findParent(int x) {
    if (parent[x] == x) return x;
    return parent[x] = findParent(parent[x]);
}

// 2. Union 연산: Union by Rank
// 트리의 높이(rank)를 비교하여 낮은 쪽을 높은 쪽 아래에 붙임
void unionNodes(int x, int y) {
    x = findParent(x);
    y = findParent(y);

	// x와 y가 같으면 이미 같은 노드 집합이다!!
    if (x != y) {
        if (rank_node[x] < rank_node[y]) {
            parent[x] = y;
        }
        else if (rank_node[x] > rank_node[y]) {
            parent[y] = x;
        }
        else {
            // 높이가 같다면 임의로 한쪽을 부모로 정하고 그쪽의 높이를 1 증가
            parent[y] = x;
            rank_node[x]++;
        }
    }
}

int main() {
    // 입출력 속도 향상
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N = 7; 
    //int M = 3;
    // cin >> N >> M;

    parent.assign(N+1,0);
    rank_node.resize(N+1,0);

    // 3. 초기화: 직접 for문을 돌려 자기 자신을 가리키게 함
    for (int i = 1; i <= N; i++) {
        parent[i] = i;
    }

    cout << "union -- " << "\n";
    unionNodes(1, 4);
    unionNodes(2, 3);
    unionNodes(3, 6);
    unionNodes(5, 7);
    unionNodes(7, 4);
    //unionNodes(4, 7);

    cout << "find -- " << "\n";
    cout << "find 1-" << findParent(1) << "\n";
    cout << "find 2-" << findParent(2) << "\n";
    cout << "find 3-" << findParent(3) << "\n";
    cout << "find 4-" << findParent(4) << "\n";
    cout << "find 5-" << findParent(5) << "\n";
    cout << "find 6-" << findParent(6) << "\n";

    cout << "parent -- " << "\n";
    for (int i = 1; i <= N; i++) {
        cout << "parent " << i << " - " << parent[i] << "\n";
    }cout << "\n";

    cout << "rank_node -- " << "\n";
    for (int i = 1; i <= N; i++) {
        cout << "rank_node " << i << " - " << rank_node[i] << "\n";
    }cout << "\n";

    return 0;
}


```

결국 parent 배열을 활용해 경로 압축을 구현한다. 그러면 parent에는 결국 어떤게 들어가냐?

union을 한 상황에서의 내 루트 노드가 들어간다. 예를들어 부모-자식 관계가 2쌍인 (1-4)와 (5-7) 이 만들어졌다고 가정하자. (코드)

그리고 union을 4와 7로 하면은 서로 부모 1과  5의 rank가 같기 때문에 임의로 하나의 노드를 부모 노드로 설정한다.

그러면 4의 부모인 1이 루트 노드로 설정되고, 4, 5, 7의 부모 노드가 1이 된다.

하지만 실제 코드, parent에서는 다르다. 구현을 어떻게 하냐면 4의 루트노드 (parent[4]) 인 1을 최종 루트노드로 설정했기 때문에 7의루트노드 (parent[7]) 인 5의 부모가 1이 되는것이다.

그렇게 되면 parent의 최종값은 `parent[1] = 1` ,  `parent[4] = 1`,  `parent[5] = 1` ,  `parent[7] = 5` 즉, parent[7] 은 5로 그대로이다. 단지 7의 루트노드가 1로 연결되는 것이다. 이게 루트 노드끼리의 연결이다. 이게 그냥 **경로 압축**을 쓰기 전의 결과이다. 

하지만, findParent에서 `return parent[x] = findParent(parent[x]);` 이렇게 수행하면 findParent[7] 을 불렀을 때 1을 반환하고 findParent 함수 내부에서 parent[7] 에 대한 값을 1로 바꿔준다. 그래서 나중에 다시 findParent(7) 을 수행하면 $O(1)$ 시간복잡도로 바로 가져올 수 있다. 이게 **경로압축**이고 실제 구현 방식이다.

그리고, rank는 높이이다. 즉, 1이 여기는 rank가 2단계까지 거쳐야 하는 노드 (1->5->7)가 있으므로 rank가 2이다. 따라서, union 할 때 작은 rank를 가진 루트 노드를 더 큰 rank 노드를 가진 루트노드에 붙여 rank를 최소화 시켜 시간복잡도를 아끼는 최적화가 바로 **Union by Rank** 이다.

기준을 rank 말고 노드 개수를 가지고 기준을 잡을 수 있다. 그것이 바로 **Union by Size**이다. 이것도 많이 쓴다.

### Union by Size

단순히 rank가 아니라 노드 개수를 가지고 루트노드를 어떤것을 선택하는 것인지 판단하는 최적화이다.

```cpp
#include <iostream>
#include <vector>
#include <algorithm>

using namespace std;

// 최대 노드 수에 맞춰 배열 크기 조절
vector<int> parent;
vector<int> sz;

// 1. Find 연산: 경로 압축(Path Compression)
// 재귀를 돌며 만나는 모든 노드의 부모를 루트로 직접 연결함
int findParent(int x) {
    if (parent[x] == x) return x;
    return parent[x] = findParent(parent[x]);
}

// 2. Union 연산: Union by Size
// 트리의 노드 개수를 기준으로 더 큰 쪽을 루트 노드로 선택함.
void unionBySize(int x, int y) {
    x = findParent(x);
    y = findParent(y);

    if (x != y) {
        // 1. 크기가 더 작은 집합을 큰 집합 밑에 붙임
        if (sz[x] < sz[y]) {
            parent[x] = y;
            sz[y] += sz[x]; // 큰 집합(y)의 크기를 업데이트
        }
        else {
            parent[y] = x;
            sz[x] += sz[y]; // 큰 집합(x)의 크기를 업데이트
        }
    }
}

int main() {
    // 입출력 속도 향상
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);

    int N = 7; 
    //int M = 3;
    // cin >> N >> M;

    parent.assign(N+1,0);
    sz.assign(N+1,1);

    // 3. 초기화: 직접 for문을 돌려 자기 자신을 가리키게 함
    for (int i = 1; i <= N; i++) {
        parent[i] = i;
    }

    cout << "union -- " << "\n";
    unionBySize(1, 4);
    unionBySize(2, 3);
    unionBySize(3, 6);
    unionBySize(5, 7);
    unionBySize(4, 7);
    //unionNodes(4, 7);

    cout << "find -- " << "\n";
    cout << "find 1-" << findParent(1) << "\n";
    cout << "find 2-" << findParent(2) << "\n";
    cout << "find 3-" << findParent(3) << "\n";
    cout << "find 4-" << findParent(4) << "\n";
    cout << "find 5-" << findParent(5) << "\n";
    cout << "find 6-" << findParent(6) << "\n";

    cout << "parent -- " << "\n";
    for (int i = 1; i <= N; i++) {
        cout << "parent " << i << " - " << parent[i] << "\n";
    }cout << "\n";

    cout << "sz -- " << "\n";
    for (int i = 1; i <= N; i++) {
        cout << "sz " << i << " - " << sz[i] << "\n";
    }cout << "\n";

    return 0;
}
```
